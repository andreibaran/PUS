Usability Test Plan Section

Test Plan for: Application Component

Persona Type Tested: Primary, Secondary, Served

Context: in a meeting room
Duration: 20 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • Assess the first interaction of the user with the application
  • Identify obstacles when trying to connect to a device
  • Evaluate the time it takes from the moment the user accesses the Device page until he manage to receive the data from the device
  • Check if the user's preferences met the options provided in the application
  • Evaluate the responsiveness of the application on normal use.
  • Test if the application is intuitive and easy to use by observing the user's behaviour
  • Receive feedback from the user with constructive observations
Scenario
  1. The user wants to monitorize another person
  2. In order to do this, he will first establish the connection between the device and his account
  3. The connection is made and the device is added in the list of the user
  4. The user receives the list with the active sensors from the monitorised person's device, together with the current values provided by each one
  5. The user can change remotely the settings from the person's device
  6. The useer can add further other devices
Tasks
  1. Hit the Device button: +10s;
  2. Connect to a new device: +2m;
  3. Access the recently added device: +20s;
  4. Request data from the device: +30s;
  5. Check the data received: +1m
  6. Change remotely a setting og the device: +2m
Total time: 6 minutes
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • Question 1: How easy was for you to connect to the device?
  • Question 2: What information from the data received you consider most valuable?
  • Question 3: Would you reccomend this application to a friend?
  • Question 4: Would you use this application again?
  • Question 5: Do you find this application of real help?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • 100% of users completed the tasks within the allocated time.
  • 80% of the users managed to perform all the tasks with no guidance
  • 95% of the users admitted that they would use the application again
  • 68% of the users said that they would pay for a licence

Summary

  • The users appreciated the accuracy of the data provided
  • Most of the users did not find necessary to chage the account default settings
  • Some of the users expressed their concern regarding the availability of the other person's device
